    About M.A.M:

    I’m currently developing a multiplayer mobile game for a university assignment, which is still in its early stages. The game is built using LibGDX and developed in Kotlin within the Linux version of Android Studio. The project currently includes a working chatroom for player communication and touchpad controls that allow users to move a character around the game world. On the backend, I’ve set up a C++ server that seamlessly communicates with clients developed in Python and Kotlin, facilitating cross-platform multiplayer functionality. While the game is still in development, I’m excited by the progress made so far and look forward to expanding on its features and gameplay elements.

    What I learnt:

    this game project that's giving me valuable hands-on experience with networking fundamentals, particularly in setting up TCP/UDP communication between clients and a server. I'm learning how to manage multiple client connections using threads to maintain smooth performance in a real-time multiplayer environment. This project is also helping me deepen my understanding of event listeners, which I’m using to handle mobile gestures and touchpad controls for character movement. As I integrate these features, I'm gaining practical knowledge in optimizing mobile games, managing network communication, and working with various programming languages and platforms. Overall, this project is strengthening my skills in mobile game development, real-time networking, and event-driven programming.
