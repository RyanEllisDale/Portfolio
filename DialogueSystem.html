Project Details

Nine-Slice Box:

Before creating the dialogue box, I needed a good way to handle the frame. To do this, I made a nine-sliced box class using the SFML C++ Library, which allows for a flexible and reusable frame. The nine-sliced box has two modes: 'stretch' and 'tiled.' In stretch mode, the segments of the sprite are resized to fit the box, while in tiled mode, the segments repeat to cover the entire area. You can see examples of both modes in the snapshots above.

Dialogue:

After creating the nine-sliced box, I used it to build the frame for the dialogue box, which allowed me to start developing the dialogue system. This system has three designs: description, sub-character, and character. The character design includes a portrait and the character's name, while the sub-character design only includes the name. The description design features just the dialogue itself. I worked on this project to get more comfortable with SFML and C++. After finishing the dialogue system, I felt ready to move on to the Retro Adventure Engine.

Highlights:

A little about the creation of this project

Text Progression::

The final challenge was implementing the ability to progress through the text and close the text box. Initially, I set it up so that the dialogue update function would run every frame, checking if the spacebar was pressed to skip to the next piece of dialogue. However, this approach didn't work well, as the project ran at 4000 frames per second, causing it to skip through all the dialogue too quickly when the user pressed the spacebar. I resolved this issue by adding a sleep mechanism to slow down the text box. I learnt a lot about inputs and timing thanks to this, as well as a bit about the main loop.

If I were to attempt this again in the future, I would like to find a more elegant solution for slowing down the dialogue, perhaps by implementing a system that checks if a certain amount of delta time has passed between spacebar presses. Additionally, I would like to enable users to navigate back through the dialogue, rather than just moving forward.

Dialogue Box:

Initially, the dialogue box was challenging because I wanted a single draw function that could render the system differently based on its display mode. This project helped me learn more about that level of polymorphism. I’m very pleased with the final designs, especially since I was able to incorporate a custom font.

If I were to redo this system in the future for more of a challenge, I would like to implement a typewriter-style effect instead of having the text appear all at once. Additionally, I would like to experiment with changing the properties of specific characters or words in the dialogue, such as highlighting a word in red, potentially using a tagging system.

Nine-Slice Box:

During this project, I learned how to effectively position, size, and render sprites. Given the potential number of sprites that the tiled mode could draw, it was crucial to optimize the nine-sliced box. I also gained a better understanding of C++ pointers, as the dialogue class uses a pointer to the nine-sliced box for its frame.

If I were to revisit this project in the future, I would add more functionality to the NineSliceBox, such as the ability to switch between the two display modes. Currently, the nine-sliced box can only be created and drawn, so enhancing its capabilities would make it more versatile.
