    About Inventory Plugin

    This custom Inventory Plugin, developed in C++ for Unreal Engine, is inspired by the inventory system from Final Fantasy VII, designed to provide developers with a flexible, ready-to-use framework for creating robust inventory mechanics. The plugin supports core features such as item pickup, gifting, and usage, with a focus on usability and customization. Players can sort their inventory based on various criteria—such as alphabetical order, quantity of items, or item type—giving designers control over how inventory data is presented and interacted with. A key feature is the item usage system, which allows each item to trigger its own unique effect. Developers can define these behaviors by binding functions or events specific to the item, giving full creative control without modifying the core inventory code. Built with Blueprint integration and scalability in mind, this plugin is suitable for a wide range of genres and teams. The project allowed me to explore Unreal’s plugin architecture, practice modular programming, and design systems that are both technically sound and intuitive for other developers to use.

    What I learnt:

    Working on this project as someone new to Unreal C++ taught me a great deal about the engine's underlying systems and how to extend them effectively. I became familiar with Unreal's class hierarchy, UObjects, ActorComponents, and how to leverage UFUNCTIONs and UPROPERTY macros to expose functionality to the editor and Blueprints. Creating a plugin helped me understand modular development, project structure, and how to separate logic cleanly for reusability. I gained practical experience managing data structures, especially for handling item storage, sorting algorithms, and inventory indexing.
